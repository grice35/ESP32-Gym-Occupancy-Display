I originally got the idea for this project around the start of my university's spring semester. The gym on my campus was overrun by the New Year's resolution crowd, making it near impossible to get a turn on some of the equipment during peak hours. In the past I had used a live crowd tracking app that took advantage of the occuspace sensors on my campus to show the busyness of various shared spaces such as the library and the recreation center, so I thought it would be a cool idea to create a display in my dorm room that pulls data from that app to make determining the best time to go to the gym easier and more convenient. I already had some exposure to working with microcontrollers with wireless capabilities from an embedded systems course I took the previous semester, so I decided to use an ESP32-C5 connected to WiFi to retrieve the occupancy data from the app and display it on an LCD screen using I2C. Working on this personal project gave me experience with HTML and JSON and how to parse specific lines and data from both. I also learned how websites and other software request data from APIs over the internet and how to do the same with an embedded device.
